Docker快速入门

简述

可以让开发者打包他们的应用以及依赖包到一个轻量级、可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。
这里必须了解2个很重要的概览：

  • 镜像
  • 容器

概览一：镜像

镜像可以理解成是光盘一样的东西，载入之后就可以直接使用了。

下载

docker的镜像可以在哪里下载呢？docker官方提供一个像github一样的仓库来存放docker的镜像。就是dockerhub。在这里可以找到一些官方的镜像或者别人修改过的第三方镜像。
docerkhub地址
至于怎么下载，很简单只要执行以下命令即可

docker pull [镜像名]

docker的镜像事实上是有版本区分的在pull的时候可以指定版本。如果不指定默认会下载最新版本。

docker pull node:10-alpine

成功下载之后，镜像就保存在本机了。

查看镜像

下载镜像之后怎么查看自己已经有的镜像呢？
执行以下命令之后可以看到本机已经有的镜像。

docker images

输出信息

REPOSITORY             TAG                 IMAGE ID            CREATED             SIZE 
node                   10-alpine           852391892b9f        2 years ago         742MB

运行镜像

接下来就可以使用我们下载的镜像了，运行镜像的命令是：

docker run --rm -it [镜像名]

即可运行镜像，可以看到终端已经进入自己的镜像环境了。
这里有两个参数

  • --rm，关闭之后自动删除容器。
  • -i，把交互界面保留着。
  • -t，分配一个虚拟的终端。

it一般都是一起出现的。
这里可以加上/bin/bash,可以进入容器的终端中。

docker run --rm -it [镜像名] /bin/bash

这时终端已经进去容器内了

// 看到容器内部分信息
root@07eac52f677e:/# cat /proc/version
Linux version 4.19.27-linuxkit (root@82332b297024) (gcc version 8.2.0 (Alpine 8.2.0)) #1 SMP Sun Mar 10 18:51:44 UTC 2019

如果要退出就执行exit即可。

概览二：容器

容器可以理解成一个提供给我们运行镜像的环境。

查看容器列表

在运行镜像之后执行

docker ps -a

可以看到当前的容器列表。
这里参数-a代表查看所有容器。如果不加a只显示正在运行的。
这里需要提到一个参数-d。

docker run  -itd [镜像名]

加上d之后运行镜像后会返回容器id。
这是时候执行docker ps -a就可以看到容器列表。

CONTAINER ID        IMAGE               COMMAND             CREATED              STATUS              PORTS               NAMES
0e2ba846e831        node:6.11           "node"              About a minute ago   Up About a minute                       focused_shannon

进入容器

用上面的命令我们可以知道自己容器的id

docker exec -it [容器id] /bin/bash

这里docker exec是一个进入容器的命令，该命令在退出容器后，容器可以继续执行。此外命令可以直接执行命令。

停止容器

假如我们需要停止容器时，只要执行docker stop [容器id]即可。

删除容器

假如我们需要停止容器时，只要执行docker rm [容器id]即可。

数据同步

如果容器中想用本机的内容进行操作可以用-v来把本机的内容挂载到容器内。

docker run -it -v /test:/soft centos /bin/bash

这里在容器的/soft中就会有本机中/test的文件。这里:前后的路径都应该用绝对路径。

网络端口映射

容器中可以运行一些网络应用，要让外部也可以访问这些应用，可以通过 -P 或 -p 参数来指定端口映射。

  • -P :是容器内部端口随机映射到主机的高端口。
  • -p : 是容器内部端口绑定到指定的主机端口。

如果用-P运行容器之后，可以用docker ps看到端口情况。

// 运行镜像
runoob@runoob:~$ docker run -d -P training/webapp python app.py
fce072cc88cee71b1cdceb57c2821d054a4a59f67da6b416fceb5593f059fc6d

// 查看看端口情况
runoob@runoob:~$ docker ps
CONTAINER ID    IMAGE               COMMAND            ...           PORTS                     NAMES
fce072cc88ce    training/webapp     "python app.py"    ...     0.0.0.0:32768->5000/tcp   grave_hopper

如果是用-p运行容器，可以指定端口映射。

runoob@runoob:~$ docker run -d -p 5000:5000 training/webapp python app.py
33e4523d30aaf0258915c368e66e03b49535de0ef20317d3f639d40222ba6bc0

另外，我们可以指定容器绑定的网络地址，比如绑定 127.0.0.1。

runoob@runoob:~$ docker run -d -p 127.0.0.1:5000:5000/udp training/webapp python app.py
6779686f06f6204579c1d655dd8b2b31e8e809b245a97b2d3a8e35abe9dcd22a

查看端口情况

docker port 命令可以让我们快捷地查看端口的绑定情况。

runoob@runoob:~$ docker port adoring_stonebraker 5000
127.0.0.1:5001

网络

上面如果我们通过端口映射可以搭建一个从容器链接本机的关系。这样就可以通过不同的容器链接到主机，然后间接链接不同的容器。这是一些简单程序的方法。如果想要让容器之间直接链接需要可以使用【网络】。

新建网络

 docker network create -d bridge 【网络名】
  • -d：参数指定 Docker 网络类型，有 bridge、overlay。其中 overlay 网络类型用于 Swarm mode，在本小节中你可以忽略它。

连接容器

// 运行第一个容器并链接到刚刚创建的网络
docker run -itd --name test1 --network test-net ubuntu /bin/bash
// 运行第二个容器并链接到刚刚创建的网络
docker run -itd --name test2 --network test-net ubuntu /bin/bash

此时进入其中一个容器

docker exec -it test1 /bin/bash

用ping来证实容器已经链接

ping test2

可以看到test1和test2已经链接了。
