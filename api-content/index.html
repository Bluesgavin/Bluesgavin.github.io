想起刚做前端的时候，刚开始工作，每天都会觉得很有收获。前端最大的特点在于可视化，也就是人们常说的所见即所得。对于刚毕业的我，可以看到自己的代码跑起来并形成页面，是一种很大成就。那时开始便会主动的去看去用一些好玩的库。 什么是学习 到了最近，我突然想要去整理一下自己的简历。却发现不知道该怎么写自己的简历。便开始想，我真的会那么多的东西了吗？好像很多的工具，我只是曾经用过而已。对工具本身并没有那么的了解。我开始想究竟"学”的定义是什么？ 举个例子，假如某天我觉得自己不太懂ES6的Symbol是什么？便随手打开百度看一看，记住了。可是工作中其实没有用到，过一段时间后，又忘了。那对于Symbol这个东西，我学会了吗？ 这里我想大胆地问，"学"的定义是什么了。经过一段时间的思考后，目前我是这样认为的，对于别人造的轮子，都只是工具。严格上来说甚至语言，再甚至系统都可以说是工具。是我们的生存工具。而对于工具，我更愿意说是"知道",而不是"学会了"。像我知道怎么用Nginx，我知道怎么用Vue。而不是"学会了"。我认为只有起码看过了源码的人，才有资格称是"学会"，至少他知道这个东西在被创造时的思路。 我们在这个岗位，要学的应该是思想。而思想，才是一个人最有价值的地方。优秀的人，他们往往是因为对某种技术有着很深的理解，让他们往更深，更正确的方向努力。等到行业发展起来之后，他们便成了当代的大神了。在想通了这一点之后。至少我应该知道，在看一些工具的时候我在做什么。当我真的需要去学习的时候，需要学的是什么。 ##未来 —— 职业定位 我是做什么的？程序员？ 我们经常说的"程序员”，其实只是一个戏称。在岗位列表上我们不会找到"程序员”这个岗位。我在和别人介绍我的工作的时候，我也会说我是一名程序员。可是自己必须要清晰自己的岗位是什么，只有这样才能明确自己往后的方向。 例如，我是一个"前端"，前端是一个岗位。面向的是以浏览器为窗口的客户。当然之后会是移动端等。这是我们应该要熟悉掌握的部分。假如往后发展呢？前端开发者的岗位，往大的说是什么？我能想到的是软件工程师，那理解起来其实就应该是我们说的"全栈"。那么"工程师"呢？ 不知道大家有没有发现，程序员的很多岗位都叫”工程师"。为什么？"工程师"是我们传统认知上，应该是干实体工程的人。像桥，路，建筑等。如果把一个工程虚拟化呢？那就是各位做的产品了。我一直都愿意把淘宝称为是一个"航母"级的工程。当我们按这个思路去想，我们就明白，我们是真正的在把一个工程从0到1得建出来，而不是在为一个模板一直"ctrl c ctrl v"。再往后是什么？架构师。很好理解，不知道大家每次做火车或者高铁，在进站时有没有留意地面上的轨道。对于这种交通工具来说，轨道的安排必须要无比准确，确保不会出现事故。工程师是造桥的人，那由谁来规划这一切，或者说，这个系统。那就是架构师。 这就是行业岗位的一个级别变化，只有在明白这一切之后。才能找到自己现在的位置，知道自己要学习的内容。当然不至是软件开发者，所有的行业，所有的岗位，都可以按这样的思路，理一理自己的职业定位。我相信大家都会有收获。对于现在这种信息爆炸的时代，一切都变化得太快了。未来的人如果想要不被时代淘汰，是注定要不断学习，不断进步的。其实也是好事，一个人在社会中，只有不断的创造价值，才会充实。 也许是到了职场瓶颈期，最近总想找点什么来做，好让自己不觉得在浪费时间。我给自己想了很多事，打算把它们当作一个个计划，希望自己可以全都坚持完成。而这其中的第一个计划，就是重新写博客。 虽然自己一直都在写技术类的博文，但这次我想尝试从生活杂事的角度来分享自己的思考。 为什么要写博客？ 在早期博客盛行的年代，粤语地区会用英语的译音把blog叫作“博lok”。最近在听podcast的时候偶然听到这个词，突然觉得倍感亲切，好像那个年代的许多事又涌现在眼前。 相信经历过互联网时代发展年代的你都一定有过写博客的经历，如163博客，QQ空间。那时候的博客就像是一个个人网站，不仅可以写文章，留下自己的联系方式，甚至可以自己修改装饰自己的主页。由于用户写的内容会用文章的形式展示，大家在记录事时会下意识像写作文一样把内容写的更加具体，就像日记一样。曾几何时，我们会通过博客来暗中关注自己留意的人，通过对方的文笔揣测他的近况。 而随着互联网的飞速发展，用户的碎片时间被利用到了极致。大部分的互联网内容从感悟思考变成了简短的资讯。微博打败了博客，短视频逐步取代长视频。网络上的大部分人似乎都在关心着媒体传达的信息，对身边人的关注变得越来越少。过去我们在博客描述一天自己经历的时候，会尝试去描述天气，会刻画自己的感情。而现在朋友圈或者微博流行的，可能只是几张图片，然后说说自己在做什么。 让老日子再过一遍，想要自己开始思想自己的生活。 写在哪里？ 这次计划遇到的第一个大问题是，我该写在哪里？过去熟悉的博客平台几乎都已经关闭了。个人网站在海外非常流行，同时也是我最喜欢的方式。我始终认为只有最简单的阅读方式才能真正发挥文字内容的价值。之前也试过自己买服务器搭博客，但没有坚持下去。也许将来某一天会再次捡起来，但目前我想我会先post在现成的平台上，以免带来过多的搭建工作。然而在看了很多平台之后，我还是没有找到满意的。朋友圈是不可能的，公众号又太多限制。微博在文字编辑上有限制，同时主页也不支持标签等功能。其他诸如简述，知乎等，都有着比较严重的主动推荐和广告问题。 最后，发现好像只有对应我程序员身份的github page比较符合我预期，因此我重启了自己的page。 写什么？ 正如开头部分讲的，这次的“重博客”只是我“Big picture”的一部分。我还给自己定了许多别的内容，大致可以分为： 工作相关的思考与技能分享 生活上的经历与感悟 阅读与观影思考 个人成长的记录 我希望通过博客这种输出方式，让自己可以把事情坚持下去，使得做的事情有价值起来。
